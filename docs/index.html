lmui

lmui (Literally My-own UI) is a minimal, semantic-first UI component library with zero external dependencies. HTML native tags are styled by default without needing any classes. Dynamic components like dialogs and dropdown use WebComponents.

I was sick of getting rug-pulled and vendor-version-locked in by Javascript frameworks and UI-libraries with numerous dependencies, endemic to the Javascript ecosystem. Not to mention, the ridiculous bloat. I want a vanilla HTML+CSS+JS UI library that I can maintain and use across my projects over the long term without being beholden to what is fashionable at the moment. So, like everyobody else, I rolled my own, yet another UI lib. My primary usecase is server-side rendered web applications.

This lib has been created with heavy LLM assistance and has a shadcn-ui-like aesthetic with zero external dependencies. I don't intend to maintain this to compete with any other framework, nor take feature requests. It's for my own use (and of course anyone else who finds it useful as-is).

What is lmui?

lmui is a minimal, semantic-first UI component library with zero dependencies. It uses modern CSS features like cascade layers, container queries, and CSS custom properties.

How do I use it?

Just include the CSS and JS files, then write semantic HTML. Most elements are styled by default without needing classes.

Is it accessible?

Yes! lmui prioritizes semantic HTML and ARIA attributes for accessibility. Keyboard navigation is fully supported.
